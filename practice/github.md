
# Установка Git


1. Обновите список пакетов:
   ```bash
   sudo apt update
   ```
2. Установите Git:
   ```bash
   sudo apt install git
   ```
3. Проверьте установку, выполнив команду:
   ```bash
   git --version
   ```
   

# Настройка Git

После установки необходимо настроить имя и email, которые будут указываться в коммитах. Выполните команды, заменив `[Your Name]` и `[your_email@example.com]` на свои данные

```bash
git config --global user.name "[Your Name]"
git config --global user.email "[your_email@example.com]"
```

Чтобы проверить текущие настройки, используйте команду:
```bash
git config --list
```
# Проверьте существование папки ~/.ssh


## Для Linux и macOS

1. **Откройте терминал.**
2. **Перейдите в домашнюю директорию** (если ещё не там):
   ```bash
   cd ~
   ```
3. **Создайте папку `.ssh`:**
   ```bash
   mkdir .ssh
   ```
4. **Установите правильные права доступа** (важно для безопасности — папка должна быть доступна только вам):
   ```bash
   chmod 700 .ssh
   ```
5. **Перейдите в созданную папку:**
   ```bash
   cd .ssh
   ```

## Для Windows

**Способ 1: через Git Bash / WSL**

1. **Запустите Git Bash или терминал WSL.**
2. **Перейдите в домашнюю директорию:**
   ```bash
   cd ~
   ```
3. **Создайте папку `.ssh`:**
   ```bash
   mkdir .ssh
   ```
4. **Настройте права доступа:**
   ```bash
   chmod 700 .ssh
   ```
5. **Перейдите в папку:**
   ```bash
   cd .ssh
   ```

**Способ 2: через командную строку (cmd) или PowerShell**

1. **Откройте cmd или PowerShell.**
2. **Создайте папку в домашней директории пользователя:**
   * В **cmd**:
     ```cmd
     mkdir "%USERPROFILE%\.ssh"
     ```
   * В **PowerShell**:
     ```powershell
     mkdir "$env:USERPROFILE\.ssh"
     ```

# Что делать после создания папки

Теперь можно **сгенерировать SSH‑ключ** и сохранить его в этой папке:

1. **Выполните команду генерации ключа** (подставьте свой email):
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com" -f ~/.ssh/id_ed25519
   ```
   Если алгоритм `ed25519` недоступен, используйте `rsa`:
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com" -f ~/.ssh/id_rsa
   ```
2. **Программа спросит пароль** (passphrase). Можно оставить поле пустым (нажмите Enter), чтобы не вводить пароль при каждом использовании ключа.
3. **Проверьте содержимое папки `.ssh`:**
   ```bash
   ls -la ~/.ssh
   ```
   Вы должны увидеть файлы:
   * `id_ed25519` или `id_rsa` — приватный ключ (не передавайте его никому!);
   * `id_ed25519.pub` или `id_rsa.pub` — публичный ключ.

# Как добавить ключ в GitHub

1. **Выведите на экран содержимое публичного ключа** и скопируйте его:
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```
   или
   ```bash
   cat ~/.ssh/id_rsa.pub
   ```
2. **Войдите в GitHub, перейдите в настройки:**
   * Кликните по аватару → **Settings**.
3. **Выберите раздел SSH and GPG keys** в меню слева.
4. **Нажмите New SSH key.**
5. **Заполните поля:**
   * **Title** — название (например, «Мой ноутбук»).
   * **Key** — вставьте скопированный ключ.
6. **Нажмите Add SSH key.**

# Проверка подключения

Убедитесь, что всё настроено верно:
```bash
ssh -T git@github.com
```
Если появится сообщение с вашим именем пользователя GitHub — всё работает.

# Важные замечания

* **Скрытость папки.** Папка `.ssh` скрыта по умолчанию из‑за точки в начале имени. Чтобы увидеть её:
   * в **Linux/macOS** используйте `ls -a`;
   * в **Windows** включите отображение скрытых файлов в проводнике.
* **Безопасность.** Никогда не передавайте приватный ключ (`id_ed25519`, `id_rsa`) третьим лицам. Храните его в защищённом месте.
* **Права доступа.** Убедитесь, что приватный ключ имеет права `600`:
  ```bash
  chmod 600 ~/.ssh/id_ed25519
  ```

# Установка и настройка SSH-ключей для GitHub

SSH-ключи обеспечивают безопасный способ аутентификации при работе с удалёнными репозиториями.  [```11```](https://www.geeksforgeeks.org/git/push-codes-from-linux-ubuntu-to-github-using-ssh-key/)

1. Сгенерируйте SSH-ключ перейдя в папку ~/.ssh "cd ~/.ssh" . Рекомендуется использовать алгоритм `ed25519`:
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```
   Где:
   * `-t ed25519`  —  тип алгоритма шифрования;
   * `-C "your_email@example.com"`  —  комментарий (обычно email), который поможет идентифицировать ключ.

   Система предложит указать путь для сохранения ключа. Нажмите `Enter`, чтобы использовать путь по умолчанию (`~/.ssh/id_ed25519`). Также можно задать парольную фразу для дополнительной защиты ключа.  [```11```](https://www.geeksforgeeks.org/git/push-codes-from-linux-ubuntu-to-github-using-ssh-key/)[```7```](https://dev.to/kellycarvalho/how-to-configure-git-on-ubuntu-and-adding-ssh-key-to-github-4h5d)

2. Добавьте приватный ключ в `ssh-agent` (менеджер ключей для SSH):
   ```bash
   eval "$(ssh-agent -s)"
   ssh-add ~/.ssh/id_ed25519
   ```

3. Скопируйте содержимое публичного ключа:
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

4. Добавьте публичный ключ в настройки аккаунта GitHub:
   * Перейдите в настройки аккаунта на GitHub: **Settings** → **SSH and GPG keys**.
   * Нажмите **New SSH key**.
   * В поле **Key** вставьте скопированный публичный ключ.
   * Укажите название ключа в поле **Title** и нажмите **Add SSH key**.  [```7```](https://dev.to/kellycarvalho/how-to-configure-git-on-ubuntu-and-adding-ssh-key-to-github-4h5d)[```9```](https://habr.com/ru/articles/755036/)

5. Проверьте подключение:
   ```bash
   ssh -T git@github.com
   ```
   При успешном подключении вы увидите сообщение о успешной аутентификации.  [```11```](https://www.geeksforgeeks.org/git/push-codes-from-linux-ubuntu-to-github-using-ssh-key/)


## Залить код на GitHub

Создайте репозиторий для своего проекта. Скопируйте адрес репозитория по ssh (название должно заканчиваться на .git)

```
git init 

git add .

git commit -m "JIRA_TASK_NUMBER"

git branch -M main

git remote add origin ССЫЛКАНАВАШРЕПОЗИТОРИЙ.git

git push origin main
```

Обычный коммит:

```
git pull origin main

git add .

git commit -m "text"

git push origin ai_service

```

Внимание! Имейте ввиду что вместо main вы указываете ту ветку на которой работаете в репозитории.
