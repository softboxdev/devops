### Практическая работа: Освоение Git на Ubuntu 

**Цель:** освоить базовые операции с Git: установку, настройку, создание репозитория, добавление и фиксацию изменений, работу с ветками.

**Время выполнения:** 120 минут.

**Требования:**
* Установленная Ubuntu (локально или в виртуальной машине).
* Доступ в интернет для установки пакетов.

#### Шаг 1. Установка и настройка Git

1. Откройте терминал (Ctrl+Alt+T).
2. Проверьте версию Git:
```bash
git --version
```

6. Проверьте настройки:
```bash
git config --list
```

#### Шаг 2. Создание локального репозитория

1. Создайте директорию для проекта и перейдите в неё через терминал:
```bash
mkdir my-first-project
cd my-first-project
```
2. Инициализируйте Git‑репозиторий:
```bash
git init
```
3. Проверьте статус репозитория:
```bash
git status
```

#### Шаг 3. Работа с файлами и коммитами

1. Создайте файл `README.md` и добавьте в него текст:
```bash
echo "# Мой первый проект" > README.md
```
2. Добавьте файл в индекс:
```bash
git add README.md
```
3. Зафиксируйте изменения (создайте коммит):
```bash
git commit -m "Добавлен README.md"
```
4. Создайте ещё один файл — `index.html`:
```bash
echo "<h1>Главная страница</h1>" > index.html
```
5. Добавьте и закоммитьте новый файл:
```bash
git add index.html
git commit -m "Добавлен index.html"
```
6. Посмотрите историю коммитов:
```bash
git log --oneline
```

#### Шаг 4. Работа с ветками

1. Создайте новую ветку `feature/new-page`:
```bash
git branch feature/new-page
```
2. Переключитесь на новую ветку:
```bash
git checkout feature/new-page
```
   *Или одной командой:*
```bash
git checkout -b feature/new-page
```
3. В ветке `feature/new-page` создайте файл `about.html`:
```bash
echo "<h1>О нас</h1>" > about.html
```
4. Добавьте и закоммитьте файл:
```bash
git add about.html
git commit -m "Добавлен about.html в новой ветке"
```
5. Вернитесь в основную ветку (`main`):
```bash
git checkout main
```
6. Слейте изменения из ветки `feature/new-page` в `main`:
```bash
git merge feature/new-page -m "Слияние ветки feature/new-page"
```
7. Удалите ветку `feature/new-page`:
```bash
git branch -d feature/new-page
```

#### Шаг 5. Проверка результатов

1. Убедитесь, что файл `about.html` появился в основной ветке:
```bash
ls -la
```
2. Посмотрите обновлённую историю коммитов:
```bash
git log --oneline --graph
```
3. Проверьте статус репозитория — он должен быть чистым:
```bash
git status
```

### Критерии оценки

* **Зачёт:** все шаги выполнены без критических ошибок, файлы созданы, коммиты зафиксированы, ветки созданы и слиты корректно. Ответы на контрольные вопросы верные.
* **Незачёт:** более двух критических ошибок в выполнении шагов, отсутствуют ключевые коммиты или ветки, ответы на контрольные вопросы неверные или неполные.

### Возможные ошибки и их решение

* **«command not found: git»** — Git не установлен. Повторите шаг 1.
* **«fatal: not a git repository»** — вы не инициализировали репозиторий или находитесь не в той директории. Проверьте текущую директорию (`pwd`) и выполните `git init`.
* **«error: pathspec '...' did not match any file(s) known to git»** — проверьте правильность написания имени файла или ветки.
* **«CONFLICT (content): Merge conflict in ...»** — конфликт слияния. Для простой работы избегайте редактирования одних и тех же файлов в разных ветках. Если конфликт возник, удалите конфликтующий файл и повторите шаги.
